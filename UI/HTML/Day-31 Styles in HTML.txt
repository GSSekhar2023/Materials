FAQ: What is the media type for styles?
Ans: You can target the styles for various media content like
        a) screen
        b) print
        c) speech

      Syntax:
         <style  type="text/css"   media="screen | print | speech | all">

         </style>

        Default media type is "all".

FAQ: What are the issues with Embedded Technique?
Ans:
       a) It is slower when compared to inline.
       b) You can't access styles accross pages.

3. Styles in External File [CSS] Cascade Style Sheets

- In this technique you can maintain styles in a separate style sheet with extention ".css"
- You can link to any page and access styles accross pages.
- Cascade is a type of arrangement, where pages can cascade over stylesheet.

Note: Issue with external style sheet is, the number of requests for page will increase which effects the page load time.

Ex:
1. Go to "src" folder
2. Add a sub folder by name "styles"
3. Add a new stylesheet into styles folder
            "effects.css"
       .container {
         border:1px solid red;
         box-shadow:2px 2px 2px red;
      }

4. Link the style sheet to any page by using "<link>" element.

    <link rel="stylesheet"  href="../src/styles/effects.css">

FAQ: If styles for any element are defined by using all 3 techniques, then which set of styles are applied to element?
Ans:
        a) If same set of effects are defined then it will apply according to priority
                1st            Inline
                2nd        Embedded
                3rd            External File

        b) If different set of effects are defined then all will apply.


                                    Style Syntax
1. If style is written in-line

        <div  style="attribute1:value; attribute2:value">

2. If style is written embedded or external file then it requires "selector".

        <style type="text/css">

            selector
            {
             color:red;
             text-align:center;
            }

        </style>

3. Attribute Selectors
4. Dynamic Pseudo Classes
5. Structural Pseudo classes
6. Element state pseudo classes
7. Element validation state pseudo classes etc..
